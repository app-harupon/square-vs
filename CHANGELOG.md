## 1. 実装済み機能

### 基本ゲームシステム
- 兵数制のターン制陣取りゲーム。歩兵/弓兵/騎兵の三すくみ、分隊・統合、地形効果(平地/森/丘/山/水/道)
- 4つの対戦規模: お手軽(7x7)/ノーマル(15x15・副将1)/大規模(30x30・副将2)/公式ルール(11x11)
- カード効果システム(盾構え・連射・強行突破・鼓舞、プレミアム: 鉄壁・狙撃・電光石火・総攻撃)
- 大将・副将のランクボーナス、名将化(ショップ、実際の課金なし)
- **精鋭兵**: 一般部隊が配置テンプレート生成時に10%の確率で精鋭化(ランク+1)。分隊・部分配置でも引き継がれる

### 3D描画(render3d.js)
- Three.jsによるフル3D地形・部隊描画。キャラクターは2Dビルボードスプライト(どの角度からも同じ見た目)
- 地形テクスチャ: 川は波模様+流れるアニメーション、山はごつごつした岩肌テクスチャ+岩塊デコレーション、道は轍・小石入りの土の小道、丘・平地は密度の異なる風にそよぐ草むら
- 空はグラデーション+雲のテクスチャ背景、影(シャドウマッピング)、明るめの照明設定
- 盤面の1本指ドラッグは地面レイキャストによる真の1:1パン(ズーム倍率に依存せず正確に追従)
- ストーリーモードの王城・砦マスでの戦闘では、敵陣奥に実際の3D城(主塔+四隅の櫓+城壁)を自動配置

### ストーリーモード(黎明の大地・国盗り合戦)
- 9x7の世界地図に10ヶ国(プレイヤーの黎明含む)を配置。隣接国への自由攻撃・同盟(同盟国は素通り可能)
- 各国に王都(🏰・落とすと総取り)と、領土3マス以上の国には前線の砦(🏯)を自動設定
- 5段階難易度(ソフト/イージー/ノーマル/ハード/ヘル)。ヘルは大国同盟による弱小国の飲み込み、ソフトは大国同士の内輪もめで無主地が発生
- 背景シミュレーション: プレイヤーが1戦するたびに他国が領土を奪い合う
- **世界情勢による兵力強化**: 領土の戦闘を5回終えるごとに、全ての国の実効兵力が10%ずつ累積強化(敵の規模・他国の侵攻確率の両方に反映)
- 降伏兵の吸収(50%/首都陥落なら100%)して予備兵力ストックに貯蓄、次の戦闘の編成に使用
- **出陣メンバー選択**: プレイヤーはノア/リオ/ガイ/セラの4キャラから大将1名・副将(所持兵力に応じた人数)を選択。各キャラ固有の顔アイコンとスキルカード(不屈の号令/百発百中/疾風怒濤/金剛の盾)を持つ。敵9ヶ国の君主にもそれぞれ固有スキルを設定
- 副将選択は予備兵力ストックから正しく差し引かれる(無限に兵が増える不具合はなし)

### UI・演出
- 起動画面: 最初の3択(CPU対戦/オンライン対戦/ストーリーモード)。CPU対戦選択時は難易度カードのスワイプカルーセルに切り替え
- スプラッシュ→メニューのフェード+せり上がりアニメーション、カードの段階的フェードイン
- カード使用時のアニメ風カットイン演出(約1.9秒、表示中は盤面操作をブロック、タップでスキップ可)
- 初回チュートリアル(スキップ可、メニューからいつでも再生可)
- CPU(相手)からの攻撃でも被害報告モーダルが表示され、閉じるまでCPUのターン進行が完全停止する

### サウンド(js/audio/sound.js)
- Web Audio APIによる完全プロシージャル生成(外部音声ファイルなし)
- 画面ごとのループBGM(メニュー/戦闘/ストーリー)、効果音(配置/移動/攻撃/カード使用/勝敗/王城陥落など)
- ミュートトグル(プロフィールに保存)

### PWA・配布
- GitHub Pagesで永続URL公開、Service Worker(network-first、キャッシュv31)でオフライン対応
- ホーム画面追加導線、難易度・進行状況はlocalStorageに保存

## 2. 今回確認した改善点

コードベース監査(パフォーマンス・オンライン対戦互換性・プロフィール移行・戻るボタン対応・未使用コード・Service Worker整合性の6項目)を実施した結果です。

| 項目 | 状態 | 重要度 |
|---|---|---|
| 3D描画パフォーマンス | ⚠️ 要対応 | 中 |
| オンライン対戦との互換性 | ✅ 問題なし | - |
| プロフィールの後方互換性 | ✅ 問題なし | - |
| Androidの戻るボタン対応 | ⚠️ 抜け漏れあり | 低〜中 |
| 未使用コード | ⚠️ 軽微 | 低 |
| Service Workerのキャッシュ一覧 | ✅ 問題なし | - |

### ⚠️ 3D描画パフォーマンス(中)
30x30の大規模バトルでは、草むら・岩・木などの地形デコレーションだけで概算**8,000〜8,500個ほどのThree.jsオブジェクト**が生成されます。低スペックのAndroid端末(RAM 2GB未満など)では描画が重くなる可能性があります。

またRenderer3Dには`dispose()`(後片付け)メソッドが存在せず、水流・草の揺れ用に常時回している`_startAmbientLoop()`の`requestAnimationFrame`ループが**一度も明示的に停止されません**。今のところRenderer3Dはアプリ全体で1個だけ使い回すシングルトン設計なので致命的なメモリリークではありませんが、将来的にレンダラーを作り直す変更を入れる場合は要注意です。

**対応案**: 大規模盤面での草むら密度を下げる(間引く)、または視界外のデコレーションを省略する。余裕があればdispose()を用意しておく。

### ⚠️ Androidの戻るボタン対応(低〜中)
`backGuardedOverlays`(戻るボタンで閉じられるモーダル一覧)に、以下の7つのモーダルが**登録されていません**:
- ストーリー難易度選択(story-difficulty-modal)
- ストーリー世界地図(story-modal)
- 国盗りマス選択(story-tile-modal)
- 出陣メンバー選択(character-select-modal)
- オンライン対戦設定(online-modal)
- 先手/後手ルーレット(turnorder-modal)
- 相手切断通知(peer-left-modal)

これらが開いている状態でAndroidの戻るボタンを押すと、モーダルを閉じずに一段飛ばして画面遷移してしまう(場合によってはアプリごと閉じる)可能性があります。これは今回新しく増えた問題ではなく、ストーリーモード自体が元々未登録だった箇所です。

**対応案**: 上記7つを`backGuardedOverlays`配列と`closeTopmostOverlay()`内のリストの両方に追加する。

### 軽微: 未使用の関数
- `js/core/rules.js`の`canAct()` — どこからも呼ばれていません
- `js/core/ai.js`の`cpuTakeTurn()` — 現在は`cpuStepTurn()`が使われており、こちらは呼ばれていません

削除しても動作に影響はありません(コード整理の余地)。

### ✅ 問題なしと確認できた点
- **オンライン対戦**: ストーリーモード専用のフィールド(`storyMap`・`storyDifficulty`など)は対戦状態の同期に一切含まれず、`isElite`フィールドも通常対戦・オンライン対戦の両方で正しく機能する汎用実装になっている
- **プロフィールの後方互換性**: 新しく追加したフィールド(`storyBattlesCompleted`・`muted`・`storyLastGeneral`など)は全て`DEFAULT_PROFILE`にデフォルト値があり、古いセーブデータを読み込んでもクラッシュしない
- **Service Workerのキャッシュ**: js/配下の全18ファイル・アイコン3種・manifest.jsonが正しくキャッシュ対象に含まれている
